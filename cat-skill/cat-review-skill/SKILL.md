# cat-review-skill

对 `cat-skill` 仓库中某个已有 skill 做一次**基础合规检查**，输出**合规判定 + 最小修改建议报告**。

## 定位

只核对硬规则清单，每项做 PASS/FAIL 二元判定，并给出最小修改建议。

不评估能力质量，不修改文件，不调用其他 skill；落地由用户决定是否采纳，或交给 `cat-create-skill` 执行写入。

## 输入

被审阅 skill 的目录路径，例如 `cat-write/zhihu-tech-ip/`。

## 审阅清单（全部为硬规则，二元判定）

复制并逐项核对，每项标 PASS 或 FAIL：

```
Frontmatter
- [ ] name 仅含 lowercase/数字/连字符，≤ 64 字符
- [ ] name 与所在目录名一致
- [ ] description 用第三人称
- [ ] description 同时包含 WHAT（能力）与 WHEN（触发条件）
- [ ] description ≤ 1024 字符

主体规范
- [ ] SKILL.md 主体 < 500 行
- [ ] 路径使用 `/` 而非 `\`
- [ ] 无时间敏感措辞（如"截至 2024 年..."）
- [ ] 术语前后一致（同一概念用同一个词）

设计合规
- [ ] 边界清晰：开篇或 description 同时声明了"做什么 + 不做什么"
- [ ] 单一职责：只声明一种核心动作（action）+ 一种作用对象（object）
- [ ] 命名自解释：name 含动作或对象信息，非 helper / utils / tool 类零信息词
- [ ] 输出规范：定义了产物形态（模板 / 报告骨架 / 字段清单 / 输出 schema 任一）
- [ ] 可独立验证:  给出了"如何判断 skill 起效"的判断点

结构规范
- [ ] 引用文件均为一级深度
- [ ] 工作流有编号步骤

仓库约定
- [ ] 位于某个 `cat-*` 顶层大类目录之下
- [ ] 子目录名 lowercase + hyphens
```

## 输出格式

```markdown
# Compliance Report: <category>/<skill-name>

## 判定
<合规 / 不合规>（FAIL 项数：N）

## 不合规项（必须修复）
- FAIL [<分组>.<项名>]: <证据片段或行号> -> <最小修改建议>

## 边缘情况（处于规则边缘，提示但不算 FAIL）
- WARN [<分组>.<项名>]: <现象> -> <提示>

## 修改 diff 建议
<针对最关键的 1–3 处给出可直接套用的替换片段>
```

报告中**禁止**出现以下能力层评价（属于迭代 skill 范围）：

- "建议丰富示例" / "示例可以更具体"
- "措辞可以更精炼" / "可以删减"
- "增强可组合性" / "改善决策路径"
- "提升 token 经济性"

若用户问到能力问题，回答"超出本 skill 范围，请使用迭代 skill"。

## 审阅原则

1. **二元判定**：每项只输出 PASS / FAIL，不允许"基本符合"等中间态
2. **规则即范围**：清单上没列的就不查，拒绝范围蔓延到能力层
3. **最小修改建议**：每条 FAIL 给出能让其变 PASS 的**最小**改动，不顺带优化
4. **边界 = 正向 + 负向**：description 或开篇必须同时写"做什么"和"不做什么"，否则视为 FAIL

## 工作流

1. 读取目标 skill 的 `SKILL.md` 及同目录附属文件
2. 逐项核对"审阅清单"，每项标 PASS / FAIL
3. 按"输出格式"产出报告
4. 报告输出后即停止；**不主动写盘、不调用其他工具落地修改**

## 验证本 skill 是否起效

执行后产物应满足：

- 报告中明确给出 PASS/FAIL 项数
- 每条 FAIL 都对应清单上的某一项（无清单外条目）
- 每条 FAIL 都附带最小修改建议
- 报告中无"丰富 / 精炼 / 优雅 / 优化"等能力层用词

## 反例（合规层面的 FAIL）

- description 第一人称（"I will help..."）-> FAIL
- description 缺少 WHAT 或 WHEN -> FAIL
- description 只讲"做什么"不讲"不做什么" -> FAIL
- 名字叫 helper / utils / tool -> FAIL
- 未定义任何产物形态（无模板/无骨架/无字段清单）-> FAIL
- 未给出"如何判断 skill 起效"的判断点 -> FAIL
- 路径用 Windows 风格 `\` -> FAIL
- name 与所在目录名不一致 -> FAIL
- 包含 "截至 2024 年..." 类时间锚 -> FAIL
