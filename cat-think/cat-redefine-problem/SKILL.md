# redefine-problem

帮助用户把一个模糊、混乱、卡住或过早收敛的问题，重新定义成更准确、更有解释力、也更容易行动的问题表述。

不替用户直接解决原问题，不输出完整战略咨询方案，不把问题过度抽象成无法执行的概念讨论。

## 输入

从用户当前表达中提取：

- 原始问题：用户现在认为自己要解决的问题
- 现状信息：已经发生了什么、影响了谁、已有约束是什么
- 用户目标：希望变成什么状态
- 已尝试动作：做过什么、哪里卡住

若关键信息缺失，先问 1-3 个澄清问题；若信息足够，直接进入工作流。

## 工作流

按以下 5 步推进，并在每一步保留可被用户确认或修正的中间产物。

1. 明确现状
   - 区分事实、判断和愿望
   - 写出当前状态、期望状态、差距和关键约束
   - 避免直接接受用户给出的第一版问题定义

2. 归因视角
   - 至少给出 2-4 个可能的归因视角
   - 常用视角包括：目标不清、资源不足、激励错位、流程断点、认知偏差、外部约束、关系冲突
   - 标注每个视角需要哪些证据支持，避免把猜测当结论

3. 深层发问
   - 针对最有解释力的归因视角提出追问
   - 优先问能改变问题定义的问题，而不是只补充细节的问题
   - 追问应帮助发现：真正约束、被忽略的利益相关方、错误前提、不可说的目标、可控与不可控边界

4. 筛选标准
   - 用标准筛选候选问题定义
   - 默认标准：更贴近事实、更可控、更能解释卡点、更能产生行动、更符合用户真实目标
   - 若用户给出特殊目标，以用户目标覆盖默认标准

5. 行动化表述
   - 把筛选后的问题改写为行动化表述
   - 表述应包含对象、情境、约束、期望结果和下一步可验证动作
   - 若需要，给出 2-3 个不同力度的版本：保守版、聚焦版、挑战版

## 输出格式

使用以下结构输出：

```markdown
## 重新定义后的问题
<一句话行动化问题表述>

## 推导过程
- 现状：<事实 / 差距 / 约束>
- 可能归因：<2-4 个视角及证据缺口>
- 深层问题：<最值得追问的 3-5 个问题>
- 筛选标准：<本次采用的标准>

## 可选表述
1. 保守版：<更稳妥的问题定义>
2. 聚焦版：<更直接的问题定义>
3. 挑战版：<更能触及深层假设的问题定义>

## 下一步行动
<一个可以在短时间内验证或推进的动作>
```

## 判断标准

一个好的重新定义结果应满足：

- 从“我要怎么办”推进到“我真正要解决的约束是什么”
- 能解释用户为什么会卡住，而不只是换一种说法
- 不依赖大量未知信息才能开始行动
- 用户看完后知道下一步该验证什么或尝试什么
- 问题表述比原问题更具体、更可控、更接近真实目标

## 验证本 skill 是否起效

执行后产物应满足：

- 明确展示“现状 → 归因视角 → 深层发问 → 筛选标准 → 行动化表述”的推导链
- 输出至少一个重新定义后的行动化问题
- 给出至少一个下一步可验证动作
- 未直接跳到解决方案清单
- 未把用户原问题原样复述为最终问题

## 反例

避免：

- 只给建议，不重新定义问题
- 只复述用户原话，不挑战问题前提
- 一上来输出解决方案列表
- 用抽象大词替代具体约束
- 在信息不足时连续追问过多问题，导致用户无法继续
