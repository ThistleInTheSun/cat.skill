# daily-log

为用户在工作区中维护"命名日志"Markdown 文件，例如 `读书日志.md`、`健身日志.md`、`工作日志.md`。

## 边界

- 做：
  1. 当用户表达"创建 XX 日志"意图 → 在工作区创建 `<主题>日志.md`
  2. 当用户表达"记录 XX 日志"意图 → 在对应文件末尾追加一行 `YYYY-MM-DD：<内容>`
- 不做：
  - 不检索、统计、聚合或可视化历史日志
  - 不修改或删除任何已有行
  - 不翻译主题词（中文保持中文，英文保持英文）
  - 不在创建文件时填入示例条目
  - 不擅自选择其他文件名或文件位置

## 触发关键词

- 创建类：`创建XX日志` / `新建XX日志` / `开一个XX日志` / `建个XX日志`
- 记录类：`记录XX日志` / `在XX日志记录` / `XX日志：今天…` / `往XX日志加一条`

## 产物形态

### 文件命名

`<主题>日志.md`

- `<主题>` 取自用户原话中的主题词，保留原语种与原字符
- 整体文件名禁止翻译、缩写、加前后缀

例：

| 用户原话 | 文件名 |
|---|---|
| "创建读书日志" | `读书日志.md` |
| "新建 gym 日志" | `gym日志.md` |
| "开一个工作日志" | `工作日志.md` |

### 文件初始内容（创建时）

新文件**只**写一个一级标题与一个空行，不塞任何示例条目：

```
# <主题>日志

```

### 单条记录格式（追加时）

每条日志独占一行：

```
YYYY-MM-DD：<内容>
```

硬要求：

- 日期使用**当天**真实日期，格式严格为 `YYYY-MM-DD`（4 位年、2 位月、2 位日，例 `2026-05-19`）
- 日期与内容之间使用**中文全角冒号** `：`（U+FF1A），不是英文 `:`，也不是空格
- `<内容>` 保留用户原话；若用户输入跨多行，统一压成一行（用空格拼接），保证一条记录占一行
- 追加到文件末尾；若文件最后一个字符不是换行，先补一个换行再追加

## 文件位置

默认放在**工作区根目录**。但记录前必须先用 Glob 搜索 `**/<主题>日志.md`：

- 命中 1 个 → 沿用该路径
- 命中多个 → 列出路径让用户选
- 未命中且当前是"记录"意图 → 询问是否先按默认路径创建

## 工作流

### A. 创建日志

1. 从用户原话中提取主题词，构造文件名 `<主题>日志.md`
2. 用 Glob 在整个工作区搜索 `**/<主题>日志.md`
3. 若已存在 → 告知用户路径并询问是否要覆盖；默认**保留原文件、不覆盖**
4. 若不存在 → 在工作区根目录写入新文件，内容为"文件初始内容"段所述格式
5. 简短回复：文件路径 + "已创建"

### B. 记录日志

1. 从用户原话中提取主题词与内容
2. 用 Glob 在整个工作区搜索 `**/<主题>日志.md`
3. 若命中多个 → 列路径让用户选；命中 1 个 → 直接用；未命中 → 询问是否先创建（确认后跳到流程 A 再回到本步骤 4）
4. 读取目标文件末尾，判断是否以换行结尾；不是就先补一个换行
5. 用今天的真实日期拼出 `YYYY-MM-DD：<内容>`，将该单行追加到文件末尾
6. 简短回复：追加到哪个文件 + 当条记录原文预览

## 示例

### 示例 1：创建

用户："创建读书日志"

→ 创建 `读书日志.md`，全部内容：

```
# 读书日志

```

### 示例 2：记录（文件已存在）

前置：`读书日志.md` 已存在，内容如上。

用户："在读书日志记录：今天读完《人月神话》第 1 章，作者强调沟通成本随团队规模平方增长"

→ 在末尾追加一行后，文件变为：

```
# 读书日志

2026-05-19：今天读完《人月神话》第 1 章，作者强调沟通成本随团队规模平方增长
```

### 示例 3：记录（文件不存在）

用户："在健身日志记录：跑步 5 公里 28 分钟"

→ Glob 未命中 `**/健身日志.md` → 询问"未找到 `健身日志.md`，是否先在工作区根目录创建？" → 用户同意 → 先按流程 A 创建，再按流程 B 追加 → 最终文件：

```
# 健身日志

2026-05-19：跑步 5 公里 28 分钟
```

## 如何判断本 skill 起效

执行后逐项核对：

- [ ] 文件名严格等于 `<主题>日志.md`，主题词未被翻译/改写
- [ ] 新建文件的非空内容只有 `# <主题>日志` 一行 + 一个空行，无任何示例条目
- [ ] 每条追加记录独占一行，且匹配正则 `^\d{4}-\d{2}-\d{2}：.+$`
- [ ] 分隔符是全角 `：`（U+FF1A），不是英文 `:`
- [ ] 日期等于操作当天的真实日期
- [ ] 历史行未被改动（diff 中除追加行外无变化）

任意一项不满足 → 本 skill 未正确执行。

## 反例

- 用英文冒号：`2026-05-19: 今天读完…` 🔴
- 日期写成 `2026/5/19` 或 `5月19日` 🔴
- 一条记录跨多行（导致后续被识别为多条） 🔴
- 把"日志"翻译成 "log" / "journal"，改写文件名 🔴
- 创建文件时塞入虚构的历史条目 🔴
- 在"记录"意图下修改了既有行而不是追加新行 🔴
- 自作主张把文件挪到 `logs/`、`journals/` 等子目录而未询问用户 🔴
