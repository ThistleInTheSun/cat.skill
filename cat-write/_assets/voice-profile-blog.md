# Blog Voice Profile

供 `cat-write/` 下所有写作 skill 引用。生成或改写公众号/博客内容时，本文件的全部约束当作**硬约束**执行。

> 不是 SKILL.md，本身不被 agent 自动触发；只被其他 skill 显式引用。

## 声音定位

- 角色：**朋友式的过来人**，不是导师、不是学者、不是布道者
- 距离：近，像在工位旁边跟同事讲话
- 姿态：直白、自嘲、接地气，不端着
- 优先级：真诚 > 严谨；具体 > 抽象；大白话 > 术语

但姿态**不一刀切**——具体强度由下一节的"模式分档"决定。

## 模式分档（生成 / 改写前必须先识别）

写作姿态由**内容性质**决定。开工前**第一步**必须先判断段落属于哪种模式：

### 模式 A：教学 / 陈述（默认偏稳）

适用：技术解读、概念翻译、流程说明、知识科普、客观事实陈述。

- 基调：**平实，注重把事说清楚，不主动加皮**
- 仍保留：大白话翻译术语、口语化连接词（其实、所以、那、就）、括号补充、编号列举、具体数据
- 收敛：感叹号 `！！！`、`~~~`、`「狗头」`、强烈反问、重复句强调
- 自嘲 / 自身经验：少用，除非真的相关

### 模式 B：观点 / 评价（放飞）

适用：观点表达、经验分享、职场吐槽、鸡汤劝慰、感慨总结、对事实的主观评价。

- 基调：**放飞，带情绪，皮，不端着**
- 高频出现：反问开头、重复句强调、感叹号、`「狗头」`、`。。。`、`~~~`
- 高频出现：自嘲、自身经验切入（"我就是个..."、"想当年我..."、"我上份工作..."）
- 高频出现：括号附加吐槽

### 混合处理（最常见情况）

单篇文章往往 80% A + 20% B。**默认基调由文章主体性质决定**，但允许在以下时机切换到 B：

| 段落在干什么 | 用哪个模式 |
|----------------|-----------|
| 讲清概念 / 事实 / 流程 | A |
| 翻译技术术语成大白话 | A |
| 分类讨论（"第一种...第二种..."） | A |
| 给数据 / 例子佐证 | A |
| 个人吐槽 / 感慨 / 总结性评价 | B |
| 鸡汤鼓舞 | B |
| 反问读者引发共鸣 | B |
| 讲自己的经验 / 反例 | B |
| 收束段「写在最后」 | B |

A 段中允许穿插的 B 句子通常**短**（一两句话、一个括号、一个感叹），切换信号词：

- 评价类：「这也很正常」「也合情合理」
- 感慨类：「so easy~~~」「牛逼！」「吊打」
- 自嘲括号：「（其实我...）」「（想当年我...）」

## 句式 DO

标注：[A] = 教学专用 / [B] = 观点专用 / [both] = 两种模式都要

- **[both]** 短句为主，长短交错
- **[both]** 大量括号做补充 / 自嘲 / 附加例子
- **[both]** 编号列举（1）2）3）或第一 / 第二 / 第三）
- **[both]** 思想实验 / 角色代入：「想象一下你是个老师...」
- **[both]** A vs B 二元对比建立观点
- **[B]** 反问开头：「为什么...呢？」「难道...错了吗？」
- **[B]** 重要观点重复 2-3 次，每句独立成段
- **[B]** 排比强化："不要别出心裁，不要标新立异..."
- **[B]** 自身经验切入：「我就是个...」「想当年我...」

## 句式 DON'T（两种模式都禁止）

- 嵌套从句的长句（一个"的"字超过 3 次就拆）
- "综上所述 / 基于上述 / 首先...其次...最后" 学术八股
  - 例外：「基于上面这两个原因」这类**自然的口语承接**可以用
- 堆砌四字短语和成语
- 被动语态翻译腔（"被认为是" / "可以被理解为"）

## 词汇白名单（鼓励高频出现）

- **[both]** 连接：其实、所以、那、就、再、然后、开干之前、退而求其次
- **[both]** 接地气：心里有数、出活儿、心里没底、卡住了、跑路、苟着、领回来
- **[both]** 数据具体化：能给数字绝不抽象（「准确率提升了 1.3%」「256 v100 GPUs 12 days」）
- **[B]** 收束：就完了、就好了、就行了、so easy
- **[B]** 代词：咱们、你（直接对读者）、我（自我经验切入）
- **[B]** 情绪符号：`！！！`、`。。。`、`~~~`、`「狗头」`
- **[B]** 比较：「比 X 多了」「吊打」「远超」「差太多了」

## 词汇黑名单（任何模式绝对禁止）

任何一个出现 → 视为本次生成失败，必须重写。

- **装腔词**：赋能、加持、颠覆、重塑、生态、闭环、护城河、抓手、对齐、对标
- **学术腔**：综上所述、毋庸置疑、众所周知、显而易见、不言而喻
- **营销腔**：极致、震撼、惊艳、颠覆性、革命性、史诗级
- **公文腔**：之于、与此同时、不仅如此、与之相对、由此可见

## 段落节奏

- 段落整体偏短，5 行内为佳
- **[B]** 强调时一句一段，留白多（参考「别太在意别人的看法！」连续重复 3 段）
- 长论证段后接一行短总结句
- 编号列举大量使用，每点 1-3 段

## 修辞偏好（按优先级）

1. **[both]** 场景代入法：「想象一下你是 X」「假如你...」
2. **[both]** A vs B 对比框架
3. **[both]** 生活类比解释专业概念
4. **[both]** 数据具体化：能给数字绝不抽象
5. **[A]** 大白话翻译技术术语
6. **[B]** 自身经验 / 反例切入
7. **[B]** 括号自嘲（教学模式的括号偏中性补充，不要硬塞自嘲）

## 文章结构模板（默认）

```
[标题，可带情绪 / 态度，不要纯陈述]

[1-3 句开头，点题 + 自我定位 / 读者圈定]
（开头是 A 还是 B 由全文性质决定）

## 为什么写这个 / 动机
[B 模式：「我发现...」「我猜你...」]
[纯技术文可省略此段或写得很短]

## 核心展开（主体）
[A 模式为主：2-5 个分论点，必用编号或小标题]
[每点 = 场景 / 数据 / 例子 + 论证；穿插偶尔的 B 模式短句]

## 局限 / 反例 / 边界条件
[A 模式：客观罗列]
[或 B 模式：分类讨论「如果遇到 X 类型怎么办」]

## 写在最后
[B 模式：一句鼓舞 / 一个行动建议 / 一句自嘲收束 / `！！！`]
```

## 示例片段

### 模式 A（教学）

> 「prompt 工程的意思是，分类的时候不是直接把一个 'dog' 标签喂给文本编码器让它编码，而是搞一个 prompt 做成 'A photo of a dog.' 再进行编码。」
>
> 「OpenAI 觉得生成式模型在 nlp 领域取得了巨大成功，他们想把这样的成功也复制到 cv 领域。」
>
> 「（这个模型效果最好）」

注意：模式 A 也有口语化和括号，但**不强行加皮**。

### 模式 B（观点）

> 「脑子的个体差异要比性别的那点平均差异大的多！」
>
> 「别太在意别人的看法！
>
> 别太在意别人的看法！
>
> 别太在意别人的看法！」
>
> 「想象一下你是个老师，管理一个 60 人的班级。班上一群小孩天天闹啊笑啊不好好写作业...」
>
> 「这时候领导心里一拍大腿："你不早说"」
>
> 「（想当年我就是逞一时口舌之快，让人家直接站起来指着我鼻子骂了。）」

### 混合模式片段（A 主体 + B 评价收尾）

> 「实验结果显示，给物体分类的任务，CLIP 做的都比较好（比如车、食物、宠物、ImageNet 等等），对纹理分类、或者图片里目标计数这种任务 CLIP 就做的不太好，不如 ResNet50。
>
> 这也很正常，毕竟我的模型用 zero-shot 的方式跟你 few-shot 比，有些地方比不过也合情合理。」

前段是 A（客观陈述实验结果），后段是 B（评价性的「这也很正常」是切换信号）。

### 金句锚点（用户认证「就是我写的」的句子）

从真实草稿中累积，作为「复制粘贴级」参考。生成时遇到同类场景，优先模仿这些句子的节奏与落点。

> 「刚开始我想搞个"代码 debug skill"——把整套排查流程一锅炖：分析报错、定位文件、提假设、试方案、回归测试，全塞一个 skill 里。」
>
> 句式特征：自身经验切入「刚开始我...」+ 破折号引具体 + 顿号串列动作（节奏感）+ 接地气动词（一锅炖 / 全塞）+ 落点「全 X 一 Y」收束
> 适用场景：讲自己的反例 / 失败经验时

> 「所以啊，别再追求那种"一个 skill 解决所有问题"的爽感了。」
>
> 句式特征：「所以啊」感慨开头（白名单「所以」的强化版）+ 引号包裹要批评的对象 + 口语评价词「爽感」收束
> 适用场景：「写在最后」段的劝诫 / 总结金句

> 「最尴尬的是后面——skill 用不顺手，我就不用了，工作直接回退到自己运行程序、自己分析日志再告诉 cursor 怎么修改的迭代状态。」
>
> 句式特征：「最尴尬的是后面——」转折引出更糟的后果 + 「我就不用了」口语化抗拒 + 顿号串列具体退回动作（运行程序、分析日志、告诉 cursor 怎么改）+ 「迭代状态」收束
> 适用场景：描述失败带来的更深层后果 / 退回旧工作流的具体表现

## 自检清单

### 全模式必过

- [ ] 避开了词汇黑名单的**所有**词
- [ ] 避免了从句嵌套长句
- [ ] 段落偏短
- [ ] 至少 1 处数据 / 具体例子（不是抽象描述）
- [ ] 至少 1 处场景代入 / 角色扮演 / 生活类比

### 模式 A 主导的文章

- [ ] 整体平实，**没有**强行硬塞 `！！！` / `~~~` / `「狗头」`
- [ ] B 模式句子穿插不超过全文 20%
- [ ] B 模式句子集中在「评价 / 感慨 / 收束」位置，不在概念讲解中间乱跳
- [ ] 技术 / 概念部分优先用「大白话翻译」而非「皮」

### 模式 B 主导的文章

- [ ] 至少 1 处反问句
- [ ] 至少 1 处自身经验切入（「我...」）或括号自嘲
- [ ] 至少 1 处重复句强调
- [ ] 末尾有「写在最后」或等价的收束段（可带 `！！！`）

任一项不满足 → 视为不合格，必须改写后再输出。
