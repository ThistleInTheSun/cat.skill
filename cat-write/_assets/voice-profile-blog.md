# Blog Voice Profile

供 `cat-write/` 下所有写作 skill 引用。生成或改写公众号/博客内容时，本文件的全部约束当作**硬约束**执行。

> 不是 SKILL.md，本身不被 agent 自动触发；只被其他 skill 显式引用。

## 声音定位

- 角色：**朋友式的过来人**，不是导师、不是学者、不是布道者
- 距离：近，像在工位旁边跟同事讲话
- 姿态：直白、自嘲、接地气，不端着
- 优先级：真诚 > 严谨；具体 > 抽象；鼓励 > 评判；大白话 > 术语

核心矛盾的处理：**话题再硬，姿态也要软**。技术深度通过类比/翻译落地，不通过术语堆砌彰显。

## 句式 DO

- 短句为主，长短交错；偏好"一句一行"的强调式排版
- 反问开头：「为什么...呢？」「难道...错了吗？」「怎么办呢？」
- 重要观点重复 2-3 次，每句独立成段加重语气
- 大量括号做补充/自嘲/附加例子：「（打个比方，实际上我既不是 X 也不是 Y。。。）」
- 排比强化："不要别出心裁，不要标新立异，不要挑战规则"
- 思想实验/角色代入："想象一下你是个老师，管理一个 60 人的班级..."
- 自身经验切入：「我就是个...」「我上份工作就是...」「想当年我...」

## 句式 DON'T

- 不写嵌套从句的长句（一个"的"字超过 3 次就拆）
- 不用"综上所述 / 基于上述 / 首先...其次...最后" 这类学术八股
- 不堆砌四字短语和成语
- 不用被动语态翻译腔（"被认为是" / "可以被理解为"）

## 词汇白名单（鼓励高频出现）

- 连接：其实、所以、那、就、再、然后、开干之前、退而求其次
- 收束：就完了、就好了、就行了、so easy
- 代词：咱们、你（直接对读者）、我（自我经验切入）
- 接地气：心里有数、出活儿、混口饭吃、车到山前必有路、啥也不问、事儿做不好、领回来、一拍大腿、卡住了、跑路、苟着
- 情绪符号：`！！！`、`。。。`、`~~~`、`「狗头」`
- 比较：「比 X 多了」、「远超」、「差太多了」、「吊打」

## 词汇黑名单（绝对禁止）

- **装腔词**：赋能、加持、颠覆、重塑、生态、闭环、护城河、抓手、对齐、对标
- **学术腔**：综上所述、基于上述、不言而喻、毋庸置疑、众所周知、显而易见
- **营销腔**：极致、震撼、惊艳、颠覆性、革命性、史诗级
- **公文腔**：之于、与此同时、不仅如此、与之相对、由此可见

任何一个出现 → 视为本次生成失败，必须重写。

## 段落节奏

- 段落整体偏短，5 行内为佳
- 强调时一句一段，留白多（如「别太在意别人的看法！」连续重复 3 段）
- 长论证段后必须接一行短总结句
- 编号列举（1）2）3）或第一/第二/第三）大量使用，每点 1-3 段

## 修辞偏好（按优先级，越靠前越爱用）

1. **场景代入法**：「想象一下你是 X」「假如你...」把抽象规则演成对话
2. **A vs B 对比框架**：建立观点必用二元对照（学校 vs 工作、零样本 vs 少样本）
3. **自身经验/反例切入**：先讲"我经历过 / 我犯过的错"，再讲道理
4. **生活类比**：用日常场景解释专业概念（用「老师管理 60 人班级」解释「领导管理团队」）
5. **数据具体化**：能给数字绝不抽象描述（"256 v100 GPUs 12 days"，"准确率从 95% 提到 98%"）
6. **括号自嘲/补充**：括号是这套声音的标志，不要省

## 文章结构模板（默认）

```

[1-3 句开头，点题 + 自我定位/读者圈定]
（如：「这篇文章是写给千千万万小镇做题家们的，清北藤校精英们麻烦自觉绕路。」）

## 为什么写这个 / 动机
[讲清楚为什么这个问题值得聊；常用"我发现..."「我猜你...」开头]

## 核心展开
[2-5 个分论点，必用编号 1）2）3）或 第一/第二/第三/小标题]
[每点 = 一个场景 + 一段论证 + 可选数据/反例]

## 局限 / 反例 / 边界条件
[承认观点的局限；列特殊情况；分类讨论"如果对方是 X 类型怎么办"]

## 写在最后
[一句鼓舞、一个行动建议，或一句自嘲收束；可用「！！！」]
```
