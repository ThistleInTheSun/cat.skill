# wechat-publish-package

为一篇已经完成的 Markdown 文稿生成「公众号发布前包」：

1. 文章末尾的标题配图 prompt
2. md2wechat 公众号排版预览页
3. 简短交付说明：prompt 追加状态、排版页面状态、需要用户手动确认的发布项

## 边界

- ✅ 做：读取已完成 `.md` 文稿，提取标题，生成配图 prompt，并追加到文稿末尾
- ✅ 做：打开 md2wechat 并粘贴追加 prompt 后的全文预览
- ❌ 不做：不改写正文，不重排内容结构，不做发布价值判断
- ❌ 不做：不调用即梦 CLI，不安装工具，不生成图片文件
- ❌ 不做：不替用户决定是否推送、群发、转发

## 输入

必须能定位到一篇已完成的 Markdown 文稿：

```yaml
md_path: <必填，已完成文稿的 .md 文件路径>
title: <可选；不填则从第一个 H1 提取，再不行用文件名>
```

如果用户只粘贴正文但没有给文件路径，先停下来要求补充目标 `.md` 路径。因为 prompt 要追加回 draft 原文件。

## 工作流

复制并跟踪：

```markdown
Task Progress:
- [ ] Step 1: 校验输入文稿路径
- [ ] Step 2: 提取标题
- [ ] Step 3: 生成配图 prompt
- [ ] Step 4: 追加 prompt 到文稿末尾
- [ ] Step 5: 打开 md2wechat 并粘贴全文预览
- [ ] Step 6: 返回交付说明
```

### Step 1: 校验输入文稿路径

读取 `md_path`，确认：

- 文件存在
- 扩展名是 `.md`
- 正文非空

任一不满足，停下要求用户补齐或修正。

### Step 2: 提取标题

标题优先级：

1. 用户显式给的 `title`
2. 文稿中的第一个一级标题：`# title`
3. md 文件名去掉扩展名

### Step 3: 生成配图 prompt

使用以下 prompt，其中 `[标题]` 替换为 Step 2 得到的标题：

```text
艺术字设计，“[标题]”几个大字，字体为手写风格，字体为无衬线体，粗体，笔画粗细统一，部分笔画进行连笔处理，线条流畅灵动，有飘逸感，字体颜色为白色，黄色装饰。英文“for freedom”在文字旁，颜色为黄色，背景为深色，有些隐约的纹理，整体风格潮流，活力，具有运动氛围感。
```

执行原则：

- 不擅自改变 prompt 中的风格词；只替换标题
- 不调用图片生成工具

### Step 4: 追加 prompt 到文稿末尾

把 prompt 追加到原 Markdown 文件末尾，格式固定为：

```markdown

---

## 配图 prompt

艺术字设计，“[标题]”几个大字，字体为手写风格，字体为无衬线体，粗体，笔画粗细统一，部分笔画进行连笔处理，线条流畅灵动，有飘逸感，字体颜色为白色，黄色装饰。英文“for freedom”在文字旁，颜色为黄色，背景为深色，有些隐约的纹理，整体风格潮流，活力，具有运动氛围感。
```

如果文稿末尾已经有 `## 配图 prompt`：

- 不重复追加
- 用新标题生成的 prompt 替换旧 prompt

### Step 5: 打开 md2wechat 并粘贴全文预览

打开：

```text
https://thistleinthesun.github.io/md2wechat/
```

把追加 prompt 后的 Markdown 全文粘贴到左侧 Markdown 输入区，让右侧显示公众号排版预览。

执行原则：

- 使用浏览器自动化工具时，直接打开页面、清空输入区、粘贴 Markdown
- 如果只能打开页面但不能自动粘贴，打开页面后明确告诉用户「页面已打开，需要手动粘贴」
- 不使用会上传正文到服务器的第三方服务；该页面是纯前端工具

### Step 6: 返回交付说明

完成后返回：

```markdown
已生成公众号发布包：

- 配图 prompt：已追加到 `<md_path>` 末尾
- 排版：md2wechat 页面已打开并粘贴全文预览
- 标题：`<title>`

发布前仍需你手动确认：是否用该 prompt 生成封面、排版是否有断行/代码块问题、是否选择提醒所有人。
```

如果某一步失败，返回失败步骤、已完成事项、下一步最小修复动作。

## 自检

交付前检查：

- [ ] 文稿末尾有且只有一个 `## 配图 prompt`
- [ ] prompt 只替换了标题，没有自行改写风格要求
- [ ] md2wechat 页面已打开
- [ ] 文稿内容已粘贴并能看到预览；若无法自动粘贴，已明确说明
- [ ] 除追加/更新配图 prompt 外，没有改写用户正文
