# audience-research

调研近期哪些话题正在获得关注，解释它们为什么受关注、背后有什么需求，以及哪些值得进入后续选题匹配。

不整理作者可写清单，不判断作者是否适合写，不推荐最终选题，不撰写文章正文。

## 输入

输入可以为空。若用户没有提供具体方向，先进行开放式近期话题扫描，并在报告中标注“空输入默认探索”。

从用户提供的信息中提取：

- 大方向：行业、领域、平台、产品类别、内容主题或人群，无输入时默认AI行业
- 目标：找热点、找痛点、找选题机会、观察趋势、扩大影响力，无输入时默认找痛点、观察趋势、扩大影响力
- 时间范围：近期、过去一周、过去一个月、季度内；若未提供，默认研究近期，无输入时默认过去一个月
- 调研范围：搜索、社交平台、社群、竞品内容、行业报告、新闻、论坛、评论区，无输入时默认以官方一手渠道为准
- 排除范围：不想研究的平台、地区、行业或敏感话题

若输入为空或缺少关键信息，优先直接推进，不要因为缺参而停止。

## 工作流

1. 确定调研范围
   - 若有用户输入，围绕输入方向调研
   - 若输入为空，选择与当前内容增长、写作、营销、AI、产品或用户上下文最相关的方向做开放扫描
   - 明确本次调研覆盖和不覆盖什么

2. 收集关注信号
   - 搜索信号：搜索趋势、常见问题、关键词、长尾疑问
   - 社交信号：高互动帖子、争议讨论、转发观点、评论区问题
   - 内容信号：近期爆文、热门视频、newsletter、播客、行业文章
   - 商业信号：产品发布、融资、招聘、销售话术、竞品内容
   - 社群信号：用户抱怨、求助、经验分享、反复出现的问题

3. 提炼近期关注话题
   - 话题必须具体，不要只写大类
   - 每个话题都要说明关注信号
   - 区分短期热闹和长期需求
   - 标注背后的痛点、情绪、动机或机会

4. 评估话题质量
   - Heat：近期热度，高 / 中 / 低
   - Durability：持续性，高 / 中 / 低
   - Pain：需求或痛点强度，高 / 中 / 低
   - Content Potential：内容展开空间，高 / 中 / 低
   - Evidence：证据强度，强 / 中 / 弱

5. 话题分级
   - 优先关注：有热度、有痛点、有持续性，值得进入 `content-idea-map`
   - 观察中：有热度但证据不足，或持续性不确定
   - 不建议追：热闹但浅、短、无明确需求，或明显不适合沉淀长期内容

6. 保存调研报告
   - 在 `cat-growth/audience-research/report/` 下创建报告
   - 文件名格式：`YYYY-MM-DD-<topic-slug>.md`
   - `<topic-slug>` 使用 lowercase letters、数字和连字符；中文主题可翻译为简短英文 slug
   - 报告必须附参考资料链接
   - 报告最后必须用一句话给出指导意见

## 输出格式

将调研报告保存到：

```text
cat-growth/audience-research/report/YYYY-MM-DD-<topic-slug>.md
```

## 质量标准

输出必须满足：

- 报告保存到 `cat-growth/audience-research/report/`
- 文件名包含日期和主题 slug
- 至少列出 5 个近期关注话题；若证据不足，说明原因
- 每个话题都要有关注信号、关注人群、背后需求和证据强度
- 至少给出 3 个“优先关注”话题或说明为什么不足 3 个
- 至少给出 3 个可转化为选题的方向
- 参考资料至少包含 2 条链接；若资料来自用户提供材料，也要列出来源说明
- 最后一节必须是“一句话指导意见”
- 输入为空时，报告必须写明默认探索假设和证据缺口

## 验证本 skill 是否起效

执行后产物应满足：

- 用户能知道近期哪些话题正在受关注
- 用户能看出每个话题为什么热、谁在关注、背后需求是什么
- 后续可以直接交给 `content-idea-map` 与作者可写清单匹配
- `report/` 目录中生成了带日期的 markdown 调研报告
- 报告包含参考资料链接和一句话指导意见

## 反例

避免：

- 把报告写成 ICP 或用户画像咨询
- 只列热点标题，不解释关注信号
- 只看热度，不判断持续性和内容展开空间
- 用作者兴趣代替外部关注信号
- 没有参考资料链接
- 没有保存报告文件
