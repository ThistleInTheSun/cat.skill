# content-retrospective

复盘内容发布后的表现，把数据和反馈转成下一轮选题、分发和作者可写清单的更新建议。

不写文章正文，不生成首次分发计划，不只看阅读量做结论。

## 输入

从用户提供的信息中提取：

- 内容对象：文章、帖子、视频、newsletter、案例、报告或活动
- 原始目标：曝光、互动、订阅、线索、成交辅助、留存或作者成长
- 发布渠道：各平台、社群、邮件、私信、合作渠道
- 数据：曝光、阅读、完读、收藏、评论、转发、关注、订阅、咨询、试用、成交辅助
- 反馈：评论、私信、销售对话、社群讨论、读者问题、反对意见
- 作者感受：是否写得顺、是否愿意继续写、是否符合作者长期方向
- 原计划：选题假设、分发假设、目标受众假设

若数据缺失，不编造结论；先区分“已知信号”和“证据缺口”。

## 工作流

1. 对齐原始目标
   - 先判断这篇内容原本要优化什么
   - 不用单一阅读量评价所有内容
   - 区分传播成功、信任成功、转化成功和学习成功

2. 整理数据和反馈
   - 定量信号：曝光、互动、订阅、线索等
   - 定性信号：评论质量、读者问题、私信、销售反馈
   - 作者信号：是否有真实表达、是否消耗过大、是否愿意延展

3. 判断假设是否成立
   - 受众假设：看到和互动的人是否是目标受众
   - 选题假设：痛点、角度、标题是否被验证
   - 分发假设：哪些渠道适配，哪些渠道无效
   - 作者假设：这个题是否真的适合作者长期写

4. 提炼可复用结论
   - 可继续写的主题
   - 需要换角度的主题
   - 不值得继续投入的主题
   - 适合沉淀为长期资产的内容
   - 适合转成销售素材或 newsletter 的内容

5. 反哺作者可写清单
   - 新增：数据证明作者适合继续写的主题
   - 升级：从 Experimental 升到 Adjacent 或 Core
   - 降级：从 Core 或 Adjacent 降级
   - 禁区：不适合作者、消耗大、反馈差或容易失真的主题

## 输出格式

```markdown
## 复盘结论
<用 3-5 句话说明这次内容表现说明了什么、下一轮应该怎么调整>

## 目标与结果
| 项目 | 内容 |
|---|---|
| 内容对象 | <内容> |
| 原始目标 | <目标> |
| 主要渠道 | <渠道> |
| 最重要结果 | <结果> |
| 最大证据缺口 | <缺口> |

## 信号整理
| 信号类型 | 观察 | 说明 | 证据强度 |
|---|---|---|---|
| 定量 | <数据> | <说明> | 强/中/弱 |
| 定性 | <反馈> | <说明> | 强/中/弱 |
| 作者 | <感受/成本> | <说明> | 强/中/弱 |

## 假设复盘
| 假设 | 是否成立 | 证据 | 下一步 |
|---|---|---|---|
| 受众假设 | 成立/部分/不成立 | <证据> | <动作> |
| 选题假设 | 成立/部分/不成立 | <证据> | <动作> |
| 分发假设 | 成立/部分/不成立 | <证据> | <动作> |
| 作者假设 | 成立/部分/不成立 | <证据> | <动作> |

## 下一轮建议
1. 继续写：<主题/角度> -> <原因>
2. 换角度：<主题/角度> -> <改法>
3. 停止或降级：<主题/角度> -> <原因>
4. 复用资产：<内容> -> <复用方式>

## 反哺作者可写清单
| 动作 | 主题 | 原层级 | 新层级 | 理由 |
|---|---|---|---|---|
| 新增/升级/降级/加入 Avoid | <主题> | <层级> | <层级> | <理由> |

## 给后续 skill 的输入
- 给 `author-topic-inventory`：<更新建议>
- 给 `content-idea-map`：<下一轮选题约束>
- 给 `content-distribution-plan`：<渠道调整>
- 给 `content-repurpose`：<值得复用的资产>
```

## 质量标准

输出必须满足：

- 先对齐原始目标，再评价表现
- 同时看定量、定性和作者信号
- 至少复盘受众、选题、分发、作者四类假设
- 必须给出对作者可写清单的更新建议
- 不把低阅读量直接等同于内容失败

## 验证本 skill 是否起效

执行后产物应满足：

- 用户知道下一轮继续写什么、换什么、停什么
- 用户知道哪些渠道和角度值得继续投入
- `author-topic-inventory` 能直接接收更新建议
- 结论能追溯到数据、反馈或明确假设

## 反例

避免：

- 只报数据，不解释学习结果
- 只看阅读量，不看目标是否达成
- 不区分内容问题、渠道问题和作者适配问题
- 没有反哺作者可写清单
